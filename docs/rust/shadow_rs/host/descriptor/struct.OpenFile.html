Expand description

Represents a POSIX file description, or a Linux struct file. An OpenFile wraps a reference to a File. Once there are no more OpenFile objects for a given File, the File will be closed. Typically this means that holding an OpenFile will ensure that the file remains open (the file’s status will not become FileState::CLOSED), but the underlying file may close itself in extenuating circumstances (for example if the file has an internal error).

Warning: If an OpenFile for a specific file already exists, it is an error to create a new OpenFile for that file. You must clone the existing OpenFile object. A new OpenFile object should probably only ever be created for a newly created file object. Otherwise for existing file objects, it won’t be clear if there are already-existing OpenFile objects for that file.

There must also not be any existing mutable borrows of the file when an OpenFile is created.
